# 5）mode

Mode用来指定当前的构建环境：production、development还是none， 设置Mode可以使用webpack内置的函数，默认为production。

选项 描述
development 设置 process.env.NODE_ENV 的值为development，开启 NamedChunksPlugin 和 NamedModulesPlugin
production 设置 process.env.NODE_ENV 的值为 production，开启 FlagDependencyUsagePlugin，FlagIncludedChunksPlugin，ModuleConcatenationPlugin，NoEmitOnErrorsPlugin，OccurrenceOrderPlugin，SideEffectsFlagPlugin 和 TerserPlugin
none 不开启任何优化选项

# 9、解析ES6和JSX

# 1）解析ES6

自从ECMA在2015年正式发布了新的JavaScript语言规范，并且每年都会发布一些JavaScript的新特性，很多浏览器无法匹配ES的更新速度，这就需要webpack这个工具将ES的新特性翻译成浏览器能够识别的ES5。

使用 babel-loader ，babel的配置文件是：.babelrc

首先，安装依赖：

npm i @babel/core @babel/preset-env babel-loader -D

在 .babelrc 文件，增加ES6的babel preset配置。

presets：是一系列plugins的集合。 plugins：可以理解为一个plugin就是一个功能。

然后再在 webpack.config.js 里面配置babel-loader。

再次 npm run build 构建项目；
